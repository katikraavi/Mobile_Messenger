## 🔗 Deep Link Architecture

```
Notification Payload
        ↓
    Deep Link
        ↓
URI Parser (merchant://invitations?tab=pending)
        ↓
Navigator.pushNamedAndRemoveUntil('/invitations')
        ↓
InvitationsScreen with tab={'pending'|'sent'}
        ↓
Pending/Sent Invitations Display
```

---

## 🔄 Backend-to-Frontend Flow

```
User A sends invite to User B
        ↓
POST /api/invites/send
        ↓
InviteService.sendInvite() creates record
        ↓
InvitesEndpoint sends notification
        ↓
FirebaseNotificationService.sendInvitationNotification()
        ↓
Firebase Cloud Messaging (FCM)
        ↓
User B's device receives notification
        ↓
forebaseMessagingBackgroundHandler (if background)
        or 
_handleNotification (if foreground)
        ↓
User taps notification
        ↓
_handleNotificationTap() extracts deepLink
        ↓
_navigateViaDeepLink('messenger://invitations?tab=pending')
        ↓
Navigator routes to InvitationsScreen
        ↓
InvitationsScreen displays pending tab
        ↓
User sees new invitation from User A
```

## 📝 Production Setup Required

To deploy push notifications in production, users will need to:

1. **Create Firebase Project**
   ```bash
   # Create project at https://console.firebase.google.com
   ```

2. **Configure Firebase for Flutter**
   ```bash
   # Install FlutterFire CLI
   dart pub global activate flutterfire_cli
   
   # Configure Firebase (generates firebase_options.dart with real credentials)
   flutterfire configure
   ```

3. **Android Setup** (`android/build.gradle`)
   - Add Google Services plugin
   - Apply plugin: `com.google.gms.google-services`

4. **iOS Setup** (`ios/Podfile`)
   - FirebaseMessaging pod will auto-configure
   - Add APNs certificate to Firebase Console

5. **Backend Setup** (`backend/.env`)
   - Add Firebase service account key
   - Configure FCM API credentials

6. **Get Device Token**
   ```dart
   final token = await PushNotificationHandler().getToken();
   // Send to backend on user login
   ```
